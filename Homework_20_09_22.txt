1  Зачем нужен GIT и какие задачи решают с его помощью?
   Git — это система управления версиями. У Git две основных задачи: первая — хранить информацию о всех изменениях в вашем коде, 
   начиная с самой первой строчки, а вторая — обеспечение удобства командной работы над кодом.


2  Что такое репозиторий GIT и где он находится?
   Репозиторий — это все файлы, находящиеся под контролем версий, вместе с историей их изменения и другой служебной информацией. 


3  Для чего используется файл .gitignore?
   gitignore нужен для скрытия файлов и папок от системы контроля версий Git.


4  Что такое "коммит"?
   коммит — это снимок локальных файлов, записанный в локальный репозиторий. 



   Укажите базовые команды в рамках работы с GIT:



5  Настройка аккаунта локального пользователя.
   git config user.name jack
   git config user.email jack@git.com

6  Создание репозитория.
   git init

7  Добавление файлов в индекс и выполнение коммита.
   git add "file name"
   git commit -m "first commit"

8  Просмотр журнала коммитов.
   git log
